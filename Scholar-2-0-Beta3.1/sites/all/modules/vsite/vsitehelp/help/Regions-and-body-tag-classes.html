Regions

The "base theme" provides a number of content regions into which scholars can move content from within the layout settings in the control panel. A region can be used or not according to preference on different pages, so custom themes should be flexible enough to respond to these changes. For instance the left column may be in use in the publications page, but not in the blog area. The right column might only be used on the site's front page, but not anywhere else.

In order to keep track of a users' preferences, CSS classes are added to the body tag to show which regions are currently in use.

Divs

Some divs are always present: #wrapper, #container, #content-wrapper, #main, #footer, #extradiv. These can be styled or left as defaults as you wish. The div #extradiv serves no function save for an additional styling div. It is otherwise empty.

The optional divs that users may add to various pages are therefore #header, #header-wrapper, #header-left, #header-main, #header-right, #navbar, #isdebar-left and #sidebar-right. To reflect which of these regions/divs is in use the body tag classes could be as follows:

body.sidebar-left, or, body.sidebar-right, or, body.both-sidebars, or, body.no-sidebars

body.header-left-main-right, or, body.header-left, or, body.header-left-main, or, body.header-main-right, or, body.header-left-right

These classes are usually strung together, for example: <body class="sidebar-left header-left-main-right">
